\section{系统设计与实现}

\subsection{项目结构}

本项目代码结构如下：

\begin{lstlisting}[language=bash]
grover_qiskit_ui_refined/
|-- app.py                    # Streamlit交互界面
|-- main.py                   # 命令行图片和CSV导出入口
|-- qiskit_grover_core.py     # Grover电路构造与statevector分析
|-- visualization.py          # Matplotlib/Plotly可视化函数
|-- requirements.txt          # Python依赖
`-- README.md                 # 项目说明
\end{lstlisting}

其中，\texttt{qiskit\_grover\_core.py}负责算法和量子态演化，
\texttt{visualization.py}负责图形绘制，\texttt{app.py}负责交互界面组织，
\texttt{main.py}用于离线生成报告图片和CSV数据。

\subsection{算法流程与状态记录}

本项目把一次Grover搜索拆分为Hadamard初始化、Oracle相位翻转和Diffusion均值倒置三个核心阶段。
程序在每个关键操作后保存当前量子态，并把这些中间状态作为后续图表的数据来源。

核心流程为：
\begin{enumerate}
  \item 初始化\(n\)个量子比特为\(|0\rangle^{\otimes n}\)；
  \item 对所有量子比特施加Hadamard门，得到均匀叠加态，并保存初始statevector；
  \item 每轮Grover迭代先施加Oracle，对目标态执行相位翻转，并保存Oracle后的statevector；
  \item 再施加Diffusion算子，完成关于平均概率幅的反演，并保存Diffusion后的statevector；
  \item 将所有中间状态传给可视化模块，绘制概率幅柱状图、概率柱状图和3D步进图。
\end{enumerate}

程序定义了\texttt{GroverStep}数据结构，用于记录每一个中间阶段：
\begin{itemize}
  \item \texttt{index}：步骤编号；
  \item \texttt{kind}：步骤类型，包括\texttt{initial}、\texttt{oracle}、\texttt{diffusion}；
  \item \texttt{iteration}：当前属于第几轮Grover迭代；
  \item \texttt{circuit}：当前量子电路副本；
  \item \texttt{statevector}：当前电路对应的精确量子态。
\end{itemize}

\subsection{运行方式}

项目提供两种运行入口。若需要启动交互式界面，可在项目根目录执行：
\begin{lstlisting}[language=bash]
streamlit run app.py
\end{lstlisting}

界面启动后，用户可以选择量子比特数、目标态和Grover迭代次数，程序会实时生成概率幅图、概率图、目标态概率曲线和3D步进图。

若需要生成报告中使用的图片和CSV数据，可在命令行中直接指定目标量子态和迭代次数：
\begin{lstlisting}[language=bash]
python main.py --n 3 --target 101 --iterations 2
\end{lstlisting}

其中，\texttt{--n}表示量子比特数，\texttt{--target}表示目标态比特串，\texttt{--iterations}表示Grover迭代次数。
程序根据这些参数构造量子电路，保存每一步statevector，并导出对应图像和数据文件。
