\subsection{迭代次数口径说明}

本文将Grover迭代次数记为\(r\)。
一次完整Grover迭代包含一次Oracle相位翻转和一次Diffusion均值倒置；
Hadamard初始化不计入Grover迭代次数，因此初始化后的状态对应\(r=0\)。

% 报告中使用了两种不同的展示范围。第一种是\(r=2\)的详细步骤展示，用于展开两轮完整Grover迭代中的中间状态，即初始化、第一次Oracle、第一次Diffusion、第二次Oracle和第二次Diffusion。第二种是目标态概率曲线中的\(r=0,1,\ldots,5\)，用于观察目标态概率随更多迭代先上升后回落的趋势。因此，表格和组图中的“迭代次数为2”表示详细展示到第二轮Grover迭代；曲线中的“最大迭代次数为5”表示额外计算到第五轮，用来说明过迭代现象。

\subsection{3量子比特目标态\texorpdfstring{\(|101\rangle\)}{|101>}示例}

选取\(n=3\)、目标态\(|101\rangle\)，并将详细步骤展示到\(r=2\)，作为静态结果图中的主要示例。此时搜索空间大小为\(N=8\)，初始每个基态概率幅为
\[
  \frac{1}{\sqrt{8}}\approx 0.353553,
\]
每个基态概率为\(1/8=0.125\)。

从表中可以看出：
\begin{itemize}
  \item Oracle步骤只改变目标态概率幅符号，不改变目标态概率；
  \item Diffusion步骤把目标态概率幅大幅推高；
  \item 两次完整迭代后，目标态概率达到约\(94.53\%\)；
  \item 如果继续迭代，目标概率会越过最佳点并下降，因此Grover算法需要选择合适的迭代次数。
\end{itemize}

\subsection{与理论结果对照}

对单目标搜索问题，Grover算法每完成一次Oracle和Diffusion组成的完整迭代，就会在目标态方向上旋转一次。对\(N=8\)，有
\[
  \theta=\arcsin\left(\frac{1}{\sqrt{8}}\right).
\]
代入理论公式：
\[
  P_1=\sin^2(3\theta)=0.78125,
\]
\[
  P_2=\sin^2(5\theta)=0.9453125.
\]
这与程序中statevector计算得到的结果一致，说明电路构造、Oracle标记、Diffusion操作以及图表数据提取是相互一致的。也就是说，当前项目虽然只实现了2到3个量子比特的玩具规模搜索，但核心演化过程与Grover算法理论相符，能够支撑课程项目对“逐步展示目标态概率幅放大”的要求。

若继续计算到\(r=5\)，目标态概率曲线会显示后续概率不再单调增大，而是出现下降。这部分结果并不是为了说明应当执行5次Grover迭代，而是为了展示小规模Grover搜索中的过迭代现象：当状态已经接近目标方向后，继续施加Grover迭代会使状态越过目标方向。

\subsection{带噪声对照的作用}

网页中加入了带噪声实验对照，但这部分不应被理解为本项目的主要算法结果。课程要求的核心是展示Oracle翻转和均值倒置后概率幅如何变化，因此理想statevector结果才是验证算法流程的主要依据。带噪声模拟只用于补充说明：当加入退相干或读出误差后，测量频率可能低于理想概率，目标态概率曲线会出现一定损耗。这个现象有助于理解真实量子电路可能受到噪声影响，但不改变当前玩具模型对Grover迭代过程的展示目标。

\subsection{2量子比特情况}

当\(n=2\)时，\(N=4\)。单目标Grover搜索在一次完整迭代后即可把目标态概率提升到1。
当\(n=3\)时，目标态概率在多次迭代中先上升后回落，程序通过目标态概率曲线记录这一过迭代现象。两种规模的结果共同说明，当前实现并不是追求大规模搜索性能，而是通过小规模、可解释的状态演化展示Grover搜索的关键机制。
