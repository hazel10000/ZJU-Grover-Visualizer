\section{项目特色与不足}

\subsection{项目特色}

\begin{enumerate}
  \item \textbf{强调中间态而非只看最终结果。} 项目在每次Oracle和Diffusion后保存statevector，适合解释振幅放大的动态过程。
  \item \textbf{同时展示概率幅和概率。} 概率幅图能显示Oracle的符号翻转，概率图能显示Diffusion后的测量概率提升。
  \item \textbf{提供3D步进视角。} 3D图把“基态--步骤--概率”同时放在一个坐标系中，便于观察目标态柱子的逐步升高。
  \item \textbf{支持交互式探索。} 用户可以切换量子比特数、目标态、迭代次数，并通过滑块逐帧检查。
  \item \textbf{保留量子电路主线。} 除图表外，界面还显示Qiskit标准电路图和交互式操作表，使算法过程能和量子门实现对应起来。
\end{enumerate}

\subsection{不足与改进方向}

本项目主要面向课堂教学，因此只支持2到3个量子比特。若扩展到更多量子比特，柱状图数量会迅速增加，3D图也会变得拥挤，需要改用筛选、聚类或只突出目标态和若干代表性非目标态的方式。另一方面，当前噪声模型只是简化的depolarizing noise和readout noise，不代表真实量子硬件的校准噪声。后续可以接入真实后端参数，或加入多目标Grover搜索的展示。
